Industry Sponsor: SightLine Applications  —  Capstone Summer–Fall 2021

A 6-person capstone team project sponsored by SightLine Applications, a real-world company that builds onboard video processors for drones, gimbals, and fixed-mount cameras. The goal was to train and deliver a Convolutional Neural Network (CNN) image classifier capable of distinguishing people, vehicles, and background in aerial footage — integrated directly into SightLine’s proprietary object detection pipeline.

The team built a custom dataset of 5,091 royalty-free images (9,000+ with augmentation), hand-curated from public datasets and self-recorded video, then preprocessed to 32×32 px grayscale PNGs. Over 1,200 training runs were executed across 9 systematic test phases, methodically sweeping epochs, learning rates, momentum, and class architecture. The best model reached ~94% accuracy on held-out test data, with real-world (“in-the-wild”) performance reaching 65% on the final 3-class model — a 9–12 percentage-point improvement over the 4-class baseline achieved by removing the ambiguous “Other” category.

Key Implementation Details

Custom Dataset Pipeline

Built a full preprocessing pipeline: sourced royalty-free images from Kaggle, Pexels, and self-recorded dashcam/drone video, then used JPEGCrops to crop objects and a custom converter.py script to resize, convert to grayscale, and normalize filenames — all automated to prevent human error across 5,000+ images.

Systematic Hyperparameter Search

Ran 9 structured test phases sweeping max_iter, base_lr, momentum, and two-phase learning rate schedules. Best configuration — max_iter 8500, lr1 10500, base_lr 0.005, lr1_base_lr 0.00015 — yielded an average accuracy of 91.97% across repeated runs.

Test Automation Script

Authored run.py to automate the full train–test cycle: it reads all hyperparameters from a single config, invokes create_trained_model.py and sl_test_model.py, logs results, and archives each run’s output folder — turning a 4-step manual process into a single command and enabling GPU-accelerated batch testing (5–10 min/run vs. 45–90 min on CPU).

3-Class vs 4-Class Architecture

Discovered that removing the catch-all “Other” class eliminated a systematic misclassification failure mode — objects with “Other” as the highest-confidence prediction were often correct in second place. Switching to a 3-class model (Background / Vehicle / Person) improved real-world accuracy by 9–12 percentage points without any additional training data.
