CS Glossary — Hash Table

Hash Table Separate Chaining File I/O OOP / Classes Dynamic Memory Recursion Pointer Arithmetic Encapsulation Bulk Source Removal


This project is a menu-driven C++ glossary application that stores and retrieves computer science definitions using a hash table with separate chaining. Terms can be loaded in bulk from a pipe-delimited text file or added one at a time by the user. Each entry stores a term name, definition, source, and citation — all heap-allocated — and is inserted at the head of the linked list at its hashed index.

The hash function converts a term into a table index by summing its ASCII character values and taking the result modulo the table size. Collisions are handled by chaining nodes in a linked list at each index, meaning multiple terms can share a slot without overwriting each other. The destructor traverses every chain in the array, deallocating each char* field and node fully before freeing the array itself.

Key Implementation Details

Hash Function

The index is computed by summing the ASCII values of every character in the key and taking the result modulo the table size (61). This distributes terms across the array based on their content, giving O(1) average-case lookup without any sorting overhead.

Separate Chaining

Each index of the array holds the head of a singly linked list. Collisions are resolved by prepending new nodes to the front of the chain — a clean O(1) insert that intentionally preserves all entries rather than overwriting them, which also allows a term to exist under multiple sources.

Bulk Source Removal

remove_all_source() scans every chain in the entire table and deletes any node whose ref field matches the given source. It handles both mid-chain removal using a previous/current pointer pair and head removal as a separate edge case, returning the total count of deleted entries.

Recursive Display

The public display_all() iterates over the array and calls a private recursive helper display_all_recursion() on each chain. The helper prints one node, recurses on next, and accumulates the count — keeping the loop and the traversal logic cleanly separated.

Code Samples

The first snippet shows the hash function and add() together. The hash function sums ASCII values and mods by the table size to produce an index, then add() allocates a new node and prepends it to the front of the chain at that index. This pair shows how the two responsibilities — computing where a term goes and physically inserting it — are kept in separate functions while working together cleanly.

table.cpp  —  hash_function() + add()

// Sum ASCII values of each character, mod by table size
int table::hash_function(char *key_value) {
  int sum = 0;
  for (int i = 0; key_value[i] != '\0'; ++i)
    sum += key_value[i];
  return sum %= table_size;
}

// Prepend new node to the front of the chain at the hashed index
int table::add(char *key_value, definition users) {
  int index = hash_function(key_value);
  node *temp = new node;
  temp->next        = hash_table[index]; // point new node to current head
  hash_table[index] = temp;              // new node becomes new head
  copy(users, index);
  return 1;
}

table.cpp  —  remove_all_source()

for (int i = 0; i < table_size; ++i) {     // scan every chain
  if (hash_table[i] && hash_table[i]->next) {
    node *previous = hash_table[i];
    node *current  = hash_table[i]->next;
    while (current) {
      if (strcmp(current->term.ref, key_value) == 0) {
        previous->next = current->next; // unlink
        delete current;
        current = previous;
        ++deleted;
      }
      previous = current;
      current  = current->next;
    }
  }
  // handle head node separately
  if (hash_table[i] &&
      strcmp(hash_table[i]->term.ref, key_value) == 0) {
    node *temp    = hash_table[i];
    hash_table[i] = hash_table[i]->next;
    delete temp;
    ++deleted;
  }
}
return deleted;

The second snippet shows remove_all_source(). Because the search key is a source rather than a term name, the function can't use the hash to narrow to a single index — it has to scan every chain in the entire table. Within each chain it uses a previous/current pointer pair to unlink mid-chain nodes, then checks the head separately as an edge case. This demonstrates the difference between a targeted hash lookup and a full-table scan, and why each requires a different traversal strategy.

Three-File Architecture

header.h

Contracts & Interfaces

Defines the definition and node structs and the full table class declaration. The single contract shared across all files.

table.cpp

Core Data Structure

Implements all table methods: constructor, destructor, hash function, add, copy, retrieve, remove_all_source, display, display_all, and the recursive display helper.

main.cpp

User Interface & File I/O

Menu loop, user input via save_users(), and bulk file loading via read_file(). All data structure logic stays fully isolated in table.cpp.
