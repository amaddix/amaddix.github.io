CS Glossary — Binary Search Tree

Binary Search Tree BST Deletion (3 Cases) In-Order Traversal Recursion OOP / Classes Dynamic Memory File I/O Wrapper/Recursive Pattern Encapsulation


This project is a menu-driven C++ glossary application that organizes computer science definitions in a Binary Search Tree — keeping terms in sorted alphabetical order at all times without any external sorting step. Each node stores a term, definition, source, and citation — all heap-allocated — and the tree is maintained purely through pointer manipulation with no standard library containers.

Every public operation follows a clean wrapper/recursive pattern: the public method accepts user-facing arguments, then delegates to a private recursive function that traverses the tree by value comparison. The destructor uses post-order recursion to fully deallocate every node and its fields without leaks, and the tree height can be queried at any time for structural insight.

Key Implementation Details

BST Insert

Insertion is handled recursively using a reference-to-pointer parameter (node*&). At each node, a strcmp determines whether to descend left or right until a NULL slot is reached, where the new node is allocated in place — preserving sorted order with no additional passes.

BST Deletion — 3 Cases

Removal handles all three structural cases: leaf nodes are deleted directly; nodes with one child are bypassed; nodes with two children find the in-order successor by traversing to the leftmost node of the right subtree via the go_left() helper, which extracts and returns that node's data before unlinking it.

Remove by Source

remove_source() performs a full tree traversal — not a directed BST search — to find and remove every node whose ref field matches the given source. This demonstrates understanding of the distinction between key-based BST lookup and full-tree scans, applying the same three-case deletion logic at each match.

In-Order Display & Height

The display() function traverses left → visit → right, producing alphabetically sorted output automatically — a property of the BST structure itself, not a sorting step. The height() function computes tree depth recursively by comparing left and right subtree heights and returning the greater value plus one.

Code Samples

The recursive insert uses a node*& parameter — a reference to a pointer — so that assigning root = new node at any depth directly wires the new node into the tree's structure. There's no need to track a parent pointer separately; the reference handles the linkage automatically.

tree.cpp  —  insert_rec()

int trees::insert_rec(node *& root, definition to_add) {
  // base case: found an empty slot — allocate here
  if (!root) {
    root = new node;
    root->term.name     = to_add.name;
    root->term.def      = to_add.def;
    root->term.ref      = to_add.ref;
    root->term.citation = to_add.citation;
    root->left = root->right = NULL;
    return 1;
  }
  // go left if key is smaller, right otherwise
  if (strcmp(to_add.name, root->term.name) < 0)
    return insert_rec(root->left,  to_add);
  else
    return insert_rec(root->right, to_add);
}

tree.cpp  —  go_left() — in-order successor

// Find leftmost node: the in-order successor.
// Copy its data out, then unlink it from the tree.
int trees::go_left(node *& root, definition & users) {
  if (!root) return 1;

  if (!root->left) {
    // reached leftmost — save its data
    users.name     = root->term.name;
    users.def      = root->term.def;
    users.ref      = root->term.ref;
    users.citation = root->term.citation;

    // bypass this node (may have a right child)
    node *temp = root;
    root = root->right;
    temp->right = temp->left = NULL;
    delete temp;
    return 1;
  }
  return go_left(root->left, users);
}

When deleting a node with two children, the BST invariant must be preserved. go_left() finds the in-order successor — the smallest value in the right subtree — by recursing left until root->left is NULL. It copies that node's data into a definition& out-parameter, then unlinks and deletes the node — leaving the tree's sorted structure intact.
