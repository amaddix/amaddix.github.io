CIFAR-10 Image Classification — MLP , CNN & K-Means

Python Machine Learning Neural Networks CNN / Deep Learning Backpropagation K-Means Clustering TensorFlow / Keras CUDA / GPU Acceleration Supervised vs. Unsupervised Hyperparameter Tuning


A group machine learning project comparing three algorithms on the CIFAR-10 dataset — 60,000 32×32 color images across 10 classes. The goal was to pit two supervised learning approaches against one unsupervised approach and measure the accuracy tradeoffs of each. My contribution covered the K-Means implementation and the project report.

The three algorithms were a hand-built Multi-Layer Perceptron (MLP) using backpropagation, a Convolutional Neural Network built with TensorFlow/Keras and accelerated with CUDA/cuDNN, and a K-Means clustering algorithm operating in the high-dimensional RGB pixel space of CIFAR's 32×32 images. Results were clear: the CNN topped out around 70–80% accuracy, the MLP peaked near 60% with optimal hyperparameters, and K-Means reached approximately 23% — expected for an unsupervised approach on a 10-class problem.

Algorithm Breakdown

Multi-Layer Perceptron — Backpropagation

Each 32×32 RGB pixel is flattened into an input vector feeding a three-layer network. Weights between input→hidden and hidden→output layers are updated via backpropagation each epoch. The baseline used 175 hidden units, learning rate 0.01, momentum 0.9 — plateauing at ~43%. Reducing momentum to 0.5 was the single best configuration, reaching ~60% accuracy. All tests showed healthy training/testing parity with no obvious overfitting. Cats and frogs (classes 3 and 6) were the hardest to classify, likely due to less distinct color profiles.

Convolutional Neural Network — TensorFlow/Keras

The CNN used three Conv2D layers (32→64→64 filters) with MaxPooling after each, followed by a Flatten, a 64-unit Dense layer, and a 10-unit output layer — 122,570 total trainable parameters. CUDA/cuDNN reduced each epoch to ~6 seconds. The network exceeded 70% testing accuracy within just 5 epochs and eventually reached ~94% training / ~68% testing — demonstrating strong learning ability but clear overfitting at later epochs. A major improvement over the MLP with no additional hand-engineering required.

K-Means — Unsupervised Clustering

K=10 centroids are randomly initialized in the high-dimensional pixel space. Each image is assigned to its nearest centroid by Euclidean distance, then centroids are recalculated as the mean of their cluster — iterated until convergence or a max iteration cap. The primary challenge was adapting the algorithm from prior 2-feature experience to CIFAR's full RGB dimensionality. Final accuracy was ~23%, which while low is competitive for a fully unsupervised 10-class problem. Training time was only 10–20 minutes compared to ~12 hours for the MLP.

Momentum was the biggest MLP lever

Varying hidden unit count (50–175) barely moved accuracy. Reducing momentum from 0.9 → 0.5 added +17 percentage points. The 50-unit model was notably faster to train with minimal accuracy cost.

CNN hit 70%+ in 5 epochs

The CNN's ability to share weights across neighboring pixels gave it a structural advantage over the fully-connected MLP. CUDA/cuDNN brought epoch time down to ~6 seconds, making iteration practical despite the 60,000-image dataset.

K-Means: dimensionality was the key challenge

Adapting K-Means from 2-feature datasets to 3,072-dimensional CIFAR images (32×32×3) required careful matrix handling. Despite the unsupervised ceiling, ~23% on 10 classes is more than double the random baseline of 10%.
