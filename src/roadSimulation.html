Road Simulation — OOP Inheritance & Dual Data Structures

OOP Inheritance Class Hierarchy Array of LLL Linked List (Path) Recursion Dynamic Memory Wrapper / Recursive Pattern Sorted Insertion Pointer-to-Pointer


This C++ project simulates driving through a city — navigating stoplights, changing lanes, and tracking your route — using a fully hand-built class hierarchy and two separate data structures working in tandem. It was built from scratch: no STL containers, no standard utilities, just raw pointers and manual memory management.

The map class models all other cars on the road as an array of sorted linked lists — one list per lane, each car ordered by position. The path class tracks every move your own car makes as a singly linked list where each new node prepends to the front. Both data structures are backed by a deep inheritance chain: lanes and location as independent base classes, vehicle inheriting from both, and node extending vehicle to become the list element used by both data structures.

Key Implementation Details

Class Hierarchy — Deep Inheritance

lanes manages which of 5 lanes a vehicle is in (bike, a–d, bus). location manages position 1–20 on the road. vehicle inherits from location and contains a lanes object, plus left/right vehicle pointers for nearby car detection. node extends vehicle and adds node* next, making it the shared list element for both data structures.

Map — Array of Sorted LLLs

The map class holds a node** array (one pointer per lane). Each lane's list is kept sorted by road position using find_spot(), a recursive function that walks previous and current pointers until it finds the right gap. move_forward() uses the wrapper/recursive pattern to increment every car's position across all lanes.

Path — History as a Prepend LLL

The path class tracks your car's history as a linked list where each new move is prepended to the front, so start always points to your most recent position. Actions are encoded as chars: 'g' = go forward, 'l' = change left, 'r' = change right. display_path() uses the wrapper/recursive pattern to print the full route.

Nearby Car Search & Stoplight Logic

map::search() takes a lane and position, recursively scans a_map[lane±1], and returns left/right vehicle pointers. These are stored directly in the vehicle's left and right fields. The stop_light class uses rand()%2 to generate a colour and a fixed freq to trigger intersections at regular position intervals.

Code Samples

add_car() shows the sorted insert logic at the core of the map data structure. It first checks if the new car belongs at the head of the list, then delegates to find_spot() — a recursive function that advances two pointers (previous and current) in tandem until it finds the correct position gap, then wires the new node in between them.

data_structure.cpp  —  add_car() + find_spot()

int map::add_car(int which_lane, int which_loc) {
  node *new_car = new node;
  new_car->set_node(which_lane, which_loc);

  // head insert: no cars yet, or new car comes first
  if (!a_map[which_lane] ||
      which_loc < a_map[which_lane]->get_pos()) {
    new_car->set_next(a_map[which_lane]);
    a_map[which_lane] = new_car;
    return 1;
  }

  // middle/tail insert — find the right gap recursively
  node *current  = a_map[which_lane];
  node *previous = current;
  current->get_next(current);
  find_spot(previous, current, which_loc);

  new_car->set_next(current);
  previous->set_next(new_car);
  return 1;
}

int map::find_spot(node *&previous, node *&current,
                   int which_loc) {
  if (!current) return 0; // reached end of list
  if (which_loc > previous->get_pos() &&
      which_loc < current->get_pos())
    return 1; // found the gap
  previous = current;
  current->get_next(current);
  find_spot(previous, current, which_loc);
}

data_structure.cpp  —  path::move_car()

int path::move_car(char action, map a_map) {
  int new_lane = start->get_lane();
  int new_pos  = start->get_pos();

  node *current = new node;

  if (action == 'g') {            // move forward
    ++new_pos;
    current->set_node(new_lane, new_pos);
  } else if (action == 'r') {     // change right
    current->set_node(new_lane + 1, new_pos);
  } else if (action == 'l') {     // change left
    current->set_node(new_lane - 1, new_pos);
  }

  // scan for left/right neighbours at new position
  current->set_pointer(a_map, new_lane, new_pos);

  // prepend to front — start always = most recent
  current->set_next(start);
  start = current;
  return 1;
}

move_car() is the core of the path data structure. Every time the user makes a move, a new node is created, its lane/position set based on the action char, its left/right neighbour pointers populated by scanning the map, and then it's prepended to the front of the list — so start always points to the car's current position and the full path history trails behind it.
