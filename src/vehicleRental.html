Rental Vehicle Manager — Doubly Linked List & Polymorphism

Doubly Linked List Polymorphism Abstract Base Class Pure Virtual Functions OOP Inheritance Dynamic Memory Wrapper / Recursive Pattern Base Pointer Storage


This C++ project is a rental vehicle management system that tracks a fleet of scooters, zip cars, and bikes using a hand-built doubly linked list. It demonstrates polymorphism through a classic base-pointer pattern: every node in the list holds a general* pointer that points to whichever derived vehicle type was added at runtime.

general is an abstract base class with pure virtual functions — calculate_total_cost(), get_repair(), get_id(), get_inuse() — forcing each derived class to implement its own cost logic and status reporting while the DLL operates entirely through the base interface. The node class wraps the base pointer and handles bi-directional traversal, while the DLL class manages the full list lifecycle using the wrapper/recursive pattern throughout.

Key Implementation Details

Abstract Base Class & Pure Virtuals

general declares calculate_total_cost(), get_repair(), get_id(), and get_inuse() as pure virtual (=0), making it impossible to instantiate directly. Each derived class — scooter ($0.05/min), zip car ($1.00/min), bike ($0.50/min) — provides its own implementation. This enforces a contract: the DLL never needs to know which vehicle it has, only that it can call these functions.

Base Pointer in node — Runtime Polymorphism

Each node holds a general* base_var. When a vehicle is added, get_derived(choice) allocates the correct derived type (new scooter, new zip, new bike) and assigns it to base_var. Every subsequent call — display(), calculate_cost(), get_repair() — goes through this pointer, dispatching to the correct derived implementation via the vtable at runtime.

Doubly Linked List — Prepend Insert

add() always prepends to the front of the list: the new node's next is set to the current head, the current head's previous is wired back to the new node, and head is updated. Both next and previous pointers are maintained on every insert so the list can be traversed in either direction at any time.

remove_vehicle() — Repair Flag Deletion

remove_vehicle() traverses the list recursively and removes any node whose vehicle has repair == true. It handles three distinct cases: head deletion (no previous pointer), mid-list deletion (rewire previous's next and advance), and no match (just traverse). The connect_nodes() helper on the node class encapsulates the pointer re-wiring logic cleanly.

Code Samples

get_derived() is where runtime polymorphism is established. Based on the user's menu choice, it allocates the correct derived type and assigns it to the general* base pointer. From this point on, every call through base_var dispatches to the right derived implementation — the node itself never needs to know which vehicle type it holds.

DLL.cpp  —  node::get_derived()

int node::get_derived(int choice) {
  // allocate correct derived type at runtime
  if      (choice == 1) base_var = new scooter;
  else if (choice == 2) base_var = new zip;
  else if (choice == 3) base_var = new bike;

  // virtual dispatch: calls derived add() via vtable
  base_var->add();
  return 1;
}

// every subsequent call goes through base_var:
void  node::display()         { base_var->display(); }
float node::calculate_cost()  { return base_var->calculate_total_cost(); }
bool  node::get_repair()       { return base_var->get_repair(); }
bool  node::get_inuse()        { return base_var->get_inuse(); }

DLL.cpp  —  DLL::add() + calculate_money()

// prepend insert — maintains both next and previous
int DLL::add(int choice) {
  node *current = new node;
  current->get_derived(choice);     // sets base_var + calls add()

  if (!head) {
    head = current;
    head->set_next(NULL);
    head->set_previous(NULL);
  } else {
    head->set_previous(current);    // wire old head back
    current->set_next(head);        // wire new node forward
    head = current;
    head->set_previous(NULL);       // new head has no previous
  }
  return 1;
}

// wrapper + recursive: sum costs across all nodes
float DLL::calculate_money(node *head) {
  if (!head) return 0;
  float current_cost = head->calculate_cost();
  head = head->get_next();
  return calculate_money(head) + current_cost;
}

add() shows the DLL prepend pattern clearly — both next and previous pointers are kept consistent on every insert. calculate_money() is a clean example of the wrapper/recursive pattern: the public wrapper calls the recursive version with head, which traverses the list accumulating each vehicle's cost through the base pointer — no switch statements, no type checks, pure virtual dispatch doing all the work.
