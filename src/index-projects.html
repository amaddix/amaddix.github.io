Phone Bill Manager

Full Stack - Java Language - Object Oriented design - Test driven development - intelliJ - android studio - REST API - maven - javadocs - jacoco



I designed and built a full-stack application that simulates a real-world phone billing system, enabling secure storage, retrieval, and analysis of customer call data.
The application allows users to submit customer and phone call transaction data, which is persisted on the server and organized by customer. Users can retrieve a complete call history for any customer and apply time-based filters to view calls within a specific date or time range, mimicking real billing and reporting workflows.
On the client side, the application maintains a local data store on the user’s smartphone, saving call transactions as they are created. Upon launching the app, users can choose to re-upload previously saved data to the server or overwrite existing records, ensuring data persistence and flexibility across sessions.
This project demonstrates practical experience with client–server communication, data persistence, file handling, and filterable query logic, while modeling how production systems manage transactional data and historical records.

Person/Vehicle Recognition Research

Machine Learning- object detection- image classifier training

REPORT PRESENTATION






6 Month - Team Research project : Myself along with, Chiharu Akiyama, Ritvik Chitram, Sanad Thapa, Tim Hall, Zhe Han


In collaboration with Leila Kaneda at SightLine Applications, a company specializing in video processing for advanced camera systems worked with SightLine to help improve the quality of their image detection services.

Our objective in this project was to create a base classifier identifying objects into 4 categories, cars, people, objects, and background. We collected a total of 4,800 un-augmented images and 9,000 augmented imaged. We found these images by looking through a number of public databases of images and videos, and by taking personal videos and cropping each person, and car. With the goal of getting an accuracy greater than 95%, we would use the data from this base classifier to help fine tune Sightline's preexisting vehicle/person classifier.
In the end, we did not succeed in getting an accuracy of 95%, however our classifier still helped Sightline find existing bugs in their software, to better improve its performance.

CIFAR-10 image recognition algorithm comparison

Python Language - Machine Learning - Object Detection - performance optimization - framework proficiency - AI algorithm comparison research - analytical mindset

REPORT PRESENTATION




Team Research project : Myself along with, Forest Pearson, Ian Wasson, Eben Weisman, Jingjing Zhao


Overview In this research-intensive project, I spearheaded a comparative analysis of supervised and unsupervised learning paradigms to solve a high-dimensional computer vision problem. Using the CIFAR-10 dataset—a benchmark collection of 60,000 color images—I developed, optimized, and benchmarked three distinct algorithmic approaches: a Multi-Layer Perceptron (MLP), a Convolutional Neural Network (CNN), and a K-Means Clustering algorithm.

Phase 1: Supervised Learning & Architecture Optimization The project began with the development of a custom Multi-Layer Perceptron (MLP). This phase focused on the mathematical foundations of deep learning, where I implemented manual backpropagation and weight-update logic. Through rigorous hyperparameter tuning, I analyzed the relationship between model complexity and convergence. By experimenting with hidden unit density and momentum values, I discovered that a leaner architecture (50 units) paired with lower momentum ($0.5$) actually mitigated oscillations and improved accuracy from a baseline of 43% to 60%. This demonstrated my ability to use empirical data to refine model performance rather than relying on default configurations.

Phase 2: High-Performance Computing with CNNs To push the boundaries of the dataset, I transitioned to a Convolutional Neural Network (CNN) built with TensorFlow and Keras. Recognizing the massive computational overhead of processing 32x32 RGB tensors, I integrated NVIDIA CUDA and cuDNN to leverage GPU acceleration. This architectural shift was transformative: Efficiency: Training time per epoch dropped to just 6 seconds, a massive improvement over the 12-hour training cycles required for the unoptimized MLP. Spatial Intelligence: By utilizing convolutional layers that reference neighboring pixel data, the model achieved a peak testing accuracy of 70%+. Analytical Insight: I identified and documented significant overfitting (94% training vs. 68% testing), providing a foundation for future implementation of dropout layers and data augmentation.

Phase 3: Unsupervised Learning & Dimensionality Finally, I implemented a K-Means Clustering algorithm to evaluate how unsupervised models handle high-dimensional raw image data. The primary challenge was the "Curse of Dimensionality"—transforming 3,072-dimensional image vectors into meaningful clusters. By optimizing matrix formations and centroid convergence logic, I achieved a stable classification rate of 23%. While expectedly lower than the supervised models, this provided critical insights into the inherent structure of the CIFAR-10 data and the limitations of heuristic clustering for complex visual tasks.

Key Technical Competencies Demonstrated Frameworks & Tools: TensorFlow, Keras, NumPy, Matplotlib. Deep Learning: CNN Architecture, Backpropagation, Hyperparameter Optimization, Gradient Descent. Hardware Acceleration: GPU Computing (NVIDIA CUDA/cuDNN). Data Science: Dimensionality Reduction, Unsupervised Clustering, Statistical Analysis of Model Loss.

Conclusion: This project demonstrates my proficiency in the full machine learning lifecycle: from data preprocessing and custom algorithm implementation to hardware-level optimization and performance benchmarking. I possess the technical agility to switch between high-level frameworks like TensorFlow and low-level mathematical implementations, ensuring I can both build and explain complex software systems.
