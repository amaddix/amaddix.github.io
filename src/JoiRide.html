Joi Ride — Road Trip Planner Web App

JavaScript Node.js Express.js EJS Templating Google Maps API Google Places API Bootstrap 5 jQuery REST APIs Heroku Deployment


A full-stack web application built for road trippers and vacationers to discover and plan stops along a route. Users enter a starting location and destination, and Joi Ride displays live turn-by-turn directions via the Google Maps Directions API. From there, users can drop a waypoint anywhere on the map and search for nearby establishments — restaurants, parks, hotels, cafes, campgrounds, and more — filtered by keyword and search radius (5–20 miles).

Built with Node.js + Express using EJS templates, jQuery for DOM interaction, and Bootstrap 5 for layout. The app integrates the Google Maps, Directions, and Places APIs together: the map auto-centers on the user's current location, address inputs use Places Autocomplete to prevent invalid entries, and each waypoint search uses nearbySearch with a dynamically read checkbox value to query the correct place type. Clicking any result opens an info window with the establishment's name and rating fetched from Places using its place_id. Deployed to Heroku.

Core Features

📍 Route & Directions

Enter a start and destination — the Directions API renders the route on the map and populates a step-by-step directions panel below. Starting address can be omitted to use current location. Address fields use Places Autocomplete to guide valid input.

📌 Waypoint Search

Drop a waypoint marker anywhere on the map. Select a keyword (Food, Hotel, Park, Cafe, Campground, Gas, Bar, Museum, Tourist Attraction, and more) and a radius — the app calls nearbySearch and displays all matching results. Multiple waypoints can be created with different keywords.

🗺️ Place Info Windows

Clicking a search result marker opens an info window populated by a Places API lookup using the result's place_id — displays the establishment name and rating. Initially the info window returned blank; solved by switching from default marker data to a dedicated Places details call.

📡 Geolocation

On load the map auto-centers on the user's current geographic location using the browser Geolocation API, making the starting point feel natural for most use cases. Users can still manually enter any starting address if preferred.

The Framework Pivot — 3 Restarts

Development began with AngularJS, then moved to React — both ran into problems rendering Google Maps API data correctly. React worked slightly better but still couldn't display what was needed. The final switch to Express + EJS succeeded, but restarting from scratch three times ate deep into available time and left a lot of work unused.

Debugging nearbySearch + Checkbox Input

Getting the nearbySearch call to read the selected checkbox value took real troubleshooting — had to correctly grab the checked value from a type="checkbox" input and pass a valid Places API type string. Once solved, keyword filtering worked cleanly and additional waypoints with different keywords could be layered on the same map.

Git Contribution — 11 commits, 769 lines

Led the majority of frontend implementation — the EJS views, jQuery event handling, Maps/Places API integration, and the waypoint/nearby search flow. Greg Hairfield handled infrastructure, routing, and deployment to Heroku.
