Workflow Graph — Adjacency List

Graph (Directed) Adjacency List Array of Vertices Linked List of Edges Pointer to Pointer OOP / Classes Dynamic Memory Linear Search


This project is a C++ graph application that models a directed workflow — a sequence of steps where each step can connect to one or more following steps. Vertices (steps) are stored in a heap-allocated array, and edges (connections) are stored as adjacency lists: each vertex holds a linked list of node pointers, each pointing directly to another vertex in the array.

The design keeps the graph compact and memory-efficient. Rather than copying vertex data into edge nodes, each node stores a raw vertex* — a pointer directly into the array — so displaying neighbors requires no lookup, just a single pointer dereference. The destructor fully traverses every adjacency list and deallocates all dynamic memory without leaks.

Key Implementation Details

Array of Vertices

The graph is backed by a dynamically allocated array of vertex structs, each holding a heap-allocated char* step name and a node* head to its adjacency list. Vertices are inserted sequentially into the next available slot, giving O(1) insertion when the position is known.

Pointer-to-Vertex Edges

Edge nodes don't copy vertex data — they store a vertex* adjacent pointing directly to the target vertex in the array. This is a deliberate design choice: it eliminates redundant storage and makes neighbor traversal a single pointer dereference rather than a search.

find_location()

find_location() performs a linear scan through the vertex array using strcmp to match by name and returns the index. Both insert_edge() and display_next() depend on this helper, keeping vertex lookup logic in one place and both callers clean.

Directed Edge Insert

insert_edge() prepends a new node to list[index1].head, setting its adjacent to &list[index2]. The edge is one-directional — only vertex1 gains a link to vertex2 — which makes this a directed graph where workflow order matters.

Code Samples

insert_edge() is the heart of the graph. It resolves both vertex names to array indices, allocates a new edge node, sets its adjacent to the address of the destination vertex in the array, and prepends it to the source vertex's adjacency list. The address-of operator (&list[index2]) is the key detail — it creates a live pointer into the array rather than a copy.

graph.cpp  —  insert_edge()

int table::insert_edge(char *vertex1, char *vertex2) {
  int index1 = find_location(vertex1);
  int index2 = find_location(vertex2);

  if (index1 < SIZE && index2 < SIZE) {
    node *temp    = new node;
    // point directly into the vertex array — no data copy
    temp->adjacent = &list[index2];
    temp->next     = list[index1].head;
    list[index1].head = temp;  // prepend to adjacency list
    return 1;
  }
  return 0; // one or both vertices not found
}

graph.cpp  —  display_next() + destructor

// one dereference to get the neighbor's name
int table::display_next(char *key_value) {
  int index = find_location(key_value);
  node *current = list[index].head;
  while (current) {
    cout << current->adjacent->step << endl;
    current = current->next;
  }
}

// destructor: free each adjacency list, then the array
table::~table() {
  for (int i = 0; i < SIZE; ++i) {
    delete [] list[i].step;
    node *cur = list[i].head;
    while (cur) {
      node *next = cur->next;
      cur->adjacent = NULL;
      delete cur;
      cur = next;
    }
  }
  delete [] list;
}

Because edge nodes hold a vertex* rather than copied data, displaying a neighbor's name is just current->adjacent->step — one pointer hop. The destructor mirrors this structure: it nulls each adjacent pointer before deleting the node (avoiding dangling pointer reads), then frees the vertex array itself — demonstrating careful, layered memory cleanup.
